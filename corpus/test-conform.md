# How to do it right
In this tutorial, you will learn how to use Ready Templates to automate responses. Firstly, you'll start with simple examples, before advancing towards using dynamic variables, which allow you to reuse the existing templates for multiple scenarios.

## Why it works
This helps set expectations early and saves readers from hitting confusing roadblocks halfway through.

Prerequisites
Typically in technical documentation, the prerequisites are usually technical based. You may expect to see something like this:

Java 13
Maven
However you can add knowledge of related services or features.

For example, if service C, requires the use A and B services to function, use these in the prerequisites section as well. 

The expanded list is:

* Java 13
* Maven
* Knowledge of Apache Camel
* Knowledge of Service A and Service B

> Crucially, each of these need to have links to the relevant tutorials, either written by you, or by the organization that owns the product or service.
